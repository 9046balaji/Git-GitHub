## 🔄 Resetting and Reverting History

- **`git reset <commit-id>`** (or **`git reset --mixed <commit-id>`**)  
  This is the default mode (`--mixed`). It moves the `HEAD` and the current branch pointer to the specified commit ID. It unstages your staged changes but keeps your actual file modifications completely unchanged in your working directory.

- **`git revert <commit-id>`**  
  Instead of rewriting history or deleting data, Git creates a brand-new commit that undoes everything introduced by that old commit. The old commit still exists in your history, and the new commit simply reverses its effects.

> [!TIP]
> Use `git revert` on shared branches (it's safe) and `git reset` on local-only branches (it rewrites history).

```diff
- git reset <commit-id>   # Rewrites history — use only locally
+ git revert <commit-id>  # Creates a new undo commit — safe for shared branches
```

## 📂 Restoring Files

- **`git restore <file-path>`**  
  If you have modified a file that was already committed, this command removes those modifications, taking the file back to its last committed state.

- **`git restore --source=<old-commit> <file-path>`**  
  Can restore deleted files, modified files, and older versions of files directly from a specific historical commit.

> [!WARNING]
> `git restore <file>` discards your uncommitted changes permanently — make sure you don't need them before running this command.

## 🧹 Cleaning Untracked Files (`git clean`)

- **`git clean -f`**  
  Removes all untracked files in the repository. The `-f` flag stands for "force," which is required to execute the cleanup. This command does **not** remove:
  - Ignored files
  - Tracked files
  - Staged files

- **`git clean -fd`**  
  Additionally removes untracked directories (folders) along with untracked files, where `-d` stands for directories.

- **`git clean -n`**  
  Shows you a dry run list of exactly which untracked files can be deleted without actually removing them yet.

- **`git clean -xfd`**  
  Deletes untracked files, untracked folders, and even ignored files from your directory.

> [!CAUTION]
> `git clean -xfd` is the nuclear option — it deletes untracked files, folders, AND ignored files. Always run `git clean -n` first to preview what will be removed.

| Command | Deletes Untracked Files | Deletes Untracked Dirs | Deletes Ignored Files | Dry Run? |
|---------|:---:|:---:|:---:|:---:|
| `git clean -n` | — | — | — | ✅ Preview |
| `git clean -f` | ✅ | ❌ | ❌ | ❌ |
| `git clean -fd` | ✅ | ✅ | ❌ | ❌ |
| `git clean -xfd` | ✅ | ✅ | ✅ | ❌ |

|---------|---------|

| `git revert <commit-id>` | Create undo commit (safe) |
| `git restore <file>` | Discard file changes |
| `git restore --source=<commit> <file>` | Restore from old commit |
| `git clean -n` | Preview deletions |
| `git clean -f` | Delete untracked files |
| `git clean -fd` | Delete untracked files + dirs |
| `git clean -xfd` | Delete everything untracked |
