# Viewing and Comparing History

## Comparing Changes

- **`git diff <commit-id>`**  
  Shows line-by-line differences, comparing your current working directory (project folder) to a specific historical commit hash.

- **`git show <commit-id>`**  
  Displays complete information about a specific commit, including commit metadata, the commit message, the author, and the date.

- **`git show <commit-id>:<file-path>`**  
  Retrieves the exact snapshot or content of a specific file from a particular commit. Note that this command displays the file's text directly and does not show a diff.

---

## Advanced History Logs (`git log`)

- **`git log --oneline`**  
  Shows your commit history in a highly compact format. Each commit is reduced to:
  - A short commit hash.
  - Only the first line of the commit message (called the commit subject or title).

- **`git log --graph`**  
  Displays an ASCII character branch and merge visualization directly beside your commits. This helps you clearly understand:
  - Where branches split apart.
  - Where merges happened.
  - Parallel development history.
  - The visual effects of a rebase.

- **`git log --author="name"`**  
  Filters your commit history to display only the commits created by a particular author. This is highly useful in collaborative team projects to inspect the contributions of specific developers.

- **`git log --since="<time/date>"`**  
  Shows the commit history from the current moment back to the specified time, day, or date only.

- **`git log --until="<time/date>"`**  
  Shows the commit history starting from that specified date all the way back to the project's very first starting date. It does not show newer commits up to today unless specified.

---

## Deep History Visualization

- **`git log --oneline --graph --all --decorate`**  
  A powerful, combined history visualization command that gives you a compact but complete overview of your entire repository structure and commit history. Here is what each part does:
  - `--oneline`: Shows a single-line summary for each commit message.
  - `--graph`: Visually represents branch splits and merges using ASCII lines.
  - `--all`: Includes the commit history from all branches instead of just your current active branch.
  - `--decorate`: Displays branch names, tags, remote pointers, and where the `HEAD` is currently attached.

---

## Tracing and Safety Nets

- **`git blame <file-path>`**  
  Performs line-level history tracing for a single file. For every single line inside that file, it displays:
  - The author who last changed the line.
  - The unique commit hash where the change happened.
  - The exact modification time and date.

- **`git reflog`**  
  Shows the movement history of your `HEAD` pointer. It acts as a safety net by tracking actions that normal logs don't show, including:
  - Branch checkouts.
  - Rebases.
  - Resets.
  - Deleted commits.
  - Branch movements.
  - Changed or amended commits.
