# Advanced Staging & Committing Commands

## Smart File Staging

- **`git add <folder>`**  
  Stages every single modified and untracked file within that specific directory. It depends strictly on which folder you run it from; only the files inside that folder will be added to the staging area.

- **`git add -A`**  
  Stages every single modified and untracked file throughout the entire project, even if you run the command from a deeply nested subfolder.

- **`git reset`**  
  Unstages a previously staged file, dropping it safely back into your local folder workspace without losing any of your written work.

---

## Advanced Committing Options

- **`git commit --amend`**  
  Instead of creating a brand-new commit, Git rewrites the previous commit and creates a new commit hash, replacing the old commit in your history. It is used to:
  - Change the last commit message.
  - Add newly staged changes directly into the last commit.
  - Replace the previous commit entirely with a new, corrected commit.

- **`git commit --all`**  
  Automatically stages all modified, tracked files right before running the commit.
  - Note: This only works for already tracked files and will **not** include newly created, untracked files.

- **`git commit -a`**  
  Provides the exact same functionality as `git commit --all`. The `-a` flag is simply a shorthand version of the `--all` command.

- **`git commit --no-verify`**  
  Tells Git to skip running safety or validation automation for this specific commit.
  - Normally, Git executes background checks like automated lint checkers, tests, formatting validation, and commit message validation.
  - By using `--no-verify`, Git completely bypasses these checks, and your commit happens immediately.

---

## Undoing Changes & Moving Timelines

- **`git reset --soft HEAD~1`**  
  Undoes your very last commit, moving the project timeline back by one step while leaving all of your modified files safely inside the staging area.

- **`git reset --hard HEAD~1`**  
  Undoes your last commit, moving the project timeline back exactly one step, and completely throws away all modifications from your local files.

- **`git reset --soft <commit-hash>` / `git reset --hard <commit-hash>`**  
  Moves your workspace directly to the designated commit using its unique commit hash number.

---

## Inspecting & Restoring Staged Changes

- **`git diff --staged`**  
  Shows the staged file changes, letting you see exactly what you modified in them before running a commit.

- **`git restore --staged <file-name>`** (or use `.` for all files)  
  Unstages your files without deleting any of your work. It removes the files from the staging area but keeps all the changes intact inside your active working directory.
